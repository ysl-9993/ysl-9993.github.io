## 📝 题目描述
- 整数数组`nums`按升序排列，数组中的值 **互不相同** 。

- 在传递给函数之前，`nums`在预先未知的某个下标`k`（`0 <= k < nums.length`）上进行了 **向左旋转**，使数组变为`[nums[k], nums[k+1], ..., nums[n-1], nums[0], nums[1], ..., nums[k-1]]`（下标 **从 0 开始** 计数）。例如，`[0,1,2,4,5,6,7]`下标`3`上向左旋转后可能变为`[4,5,6,7,0,1,2]`。

- 给你 **旋转后** 的数组`nums`和一个整数`target`，如果`nums`中存在这个目标值`target`，则返回它的下标，否则返回`-1`。

- 你必须设计一个时间复杂度为`O(log n)`的算法解决此问题。


## 🤔 解题思路
### 哈希表（O(n)时间复杂度）
- 遍历数组，将每个元素的值和下标以键值对的形式存储到哈希表中。

### 二分查找（O(log n)时间复杂度）
- 二分查找是针对有序数组的查找算法，时间复杂度为`O(log n)`。
- 本题中数组被分为有序的两部分，分别为`[0, k-1]`和`[k, n-1]`，可以对二分查找进行改造。
- 先找到`mid`，若`nums[mid] != target`，则通过和`nums[0]`比较，判断`mid`在左半部分还是右半部分。
    - 若`nums[mid] >= nums[0]`，则`mid`在左半部分。
        - 若`nums[mid] > target`，则`target`可能在左半部分，也可能在右半部分。
            - 若`nums[0] <= target`，则`target`可能在左半部分。
            - 若`nums[0] > target`，则`target`一定不在左半部分，因为左半部分最小的数都比`target`大，`target`可能在右半部分。
        - 若`nums[mid] < target`，则`target`可能在左半部分，移动`l`指针继续二分查找。
    - 若`nums[mid] < nums[0]`，则`mid`在右半部分。
        - 若`nums[mid] >= target`，则`target`可能在右半部分，移动`r`指针继续二分查找。
        - 若`nums[mid] < target`，则`target`可能在右半部分，也可能在左半部分。
            - 若`nums[-1] >= target`，则`target`可能在右半部分。
            - 若`nums[-1] < target`，则`target`一定不在右半部分，因为右半部分最大的数都比`target`小，`target`可能在左半部分。


## 🧑‍💻 题解
### 哈希表（O(n)时间复杂度，O(n)空间复杂度）
```python
class Solution:
    def search(self, nums: List[int], target: int) -> int:
        cache = dict()
        for i in range(len(nums)):
            cache[nums[i]] = i
        if target in cache:
            return cache[target]
        else:
            return -1
```

### 二分查找（O(log n)时间复杂度，O(1)空间复杂度）
```python
class Solution:
    def search(self, nums: List[int], target: int) -> int:
        l = 0
        r = len(nums) - 1
        while l <= r:
            mid = (l + r) // 2
            if nums[mid] == target:
                return mid
            if nums[mid] >= nums[0]:
                # mid在左半部分
                if nums[mid] > target:
                    if nums[0] <= target:
                        r = mid - 1
                    else:
                        # 答案可能在右半部分
                        l = mid + 1
                else:
                    l = mid + 1
            else:
                # mid 在右半部分
                if nums[mid] < target:
                    if nums[-1] >= target:
                        l = mid + 1
                    else:
                        # 答案可能在左半部分
                        r = mid - 1
                else:
                    r = mid -1
        return -1
```
