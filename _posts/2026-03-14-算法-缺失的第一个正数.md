## 📝 题目描述
给你一个未排序的整数数组`nums`，请你找出其中没有出现的最小的正整数。

请你实现时间复杂度为`O(n)`并且只使用常数级别额外空间的解决方案。

## 🤔 解题思路
### 不考虑O(1)空间复杂度的情况
直接遍历`nums`，存入哈希表中，再从`1`开始遍历，判断是否在哈希集合中，满足时间复杂度`O(n)`。

### 考虑O(1)空间复杂度的情况
#### 思路
- 在`O(1)`空间复杂度的要求下，还要满足`O(n)`的时间复杂度，只能考虑在原数组上操作。
- 如果`nums`的长度为`n`，那么缺失的第一个正数只能在`[1, n+1]`之间，因为如果`nums`中包含`[1, n]`之间的所有整数，那么缺失的第一个正数为`n+1`，否则缺失的第一个正数在`[1, n]`之间。换句话说，如果`num`中出现负数，那么答案一定不是`n+1`，而是在`[1, n]`之间。
- 答案区间恰好与数组长度相等，因此可以将数组的值的问题转化为数组索引的问题。

#### 步骤
1. 先将`nums`中所有原本的负数和0变为`n+1`（因为如果存在负数或0，答案一定在`[1, n]`之间，所以变成`n+1`不会影响答案），避免影响标记。
2. 然后遍历`nums`，对所有在`[1, n]`之间的整数`x`, 将`nums[x-1]`标记为负数（表示`x`在`nums`中出现过）。
3. 最后遍历`nums`，找到第一个正数`nums[i]`，那么`i+1`就是缺失的第一个正数。

至此巧妙地将`nums`的值转换成了`nums`的索引，通过索引即可找到缺失的第一个正数。

## 🧑‍💻 题解（O(n)空间复杂度）
```python
class Solution:
    def firstMissingPositive(self, nums: List[int]) -> int:
        cache = set()
        for num in nums:
            if num > 0:
                cache.add(num)
        i = 1
        while i in cache:
            i += 1
        return i
```

## 🧑‍💻 题解（O(1)空间复杂度）
```python
class Solution:
    def firstMissingPositive(self, nums: List[int]) -> int:
        for i in range(len(nums)):
            # 先将原本的负数和0移出影响范围
            if nums[i] <= 0:
                nums[i] = len(nums) + 1
        for i in range(len(nums)):
            # 再对原数组进行改造 注意加abs找nums[i]原本的值
            if abs(nums[i]) >= 1 and abs(nums[i]) <= len(nums):
                # 将nums[i]的值作为索引 标记第nums[i]-1个元素为负数 表示数字nums[i]出现过
                if nums[abs(nums[i]) - 1] > 0:
                    # 如果还没有被标记 否则跳过
                    nums[abs(nums[i]) - 1] = - nums[abs(nums[i]) - 1]
        for i in range(len(nums)):
            # 最后找第一个没有被标记的数字的索引
            if nums[i] > 0:
                return i + 1
        # 如果全部被标记 答案就是n+1
        return len(nums) + 1
```
## 👀 易错点
```python
if abs(nums[i]) >= 1 and abs(nums[i]) <= len(nums):
    if nums[abs(nums[i]) - 1] > 0:
        nums[abs(nums[i]) - 1] = - nums[abs(nums[i]) - 1]
```
> 一定要注意加`abs()`，因为`nums[i]`可能已经被标记为负数了，而我们还需要`nums[i]`的原值找新的索引。
