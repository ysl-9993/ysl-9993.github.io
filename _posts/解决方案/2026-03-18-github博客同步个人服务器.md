## 背景
因为我的个人博客是部署在`github pages`上的，但是有时会因为网络问题导致博客无法访问，所以计划将博客同步到个人服务器，确保博客访问的稳定性。

## 步骤
### 1. 创建SSH密钥对
- 打开终端，执行命令生成密钥对
    ```bash
    ssh-keygen
    ```
- 一直回车，直到显示
    ```bash
    xxxxx already exists
    ```
- 查看生成的密钥对（xxxx替换成实际密钥文件名）
    ```bash
    # 公钥
    cat ~/.ssh/id_xxxx.pub

    # 私钥
    cat ~/.ssh/id_xxxx
    ```

### 2. 在github仓库添加SSH密钥
- 进入仓库`Settings`->`Secrets and variables`->`Actions`   
- 点击`New repository secret`，添加两个密钥
    ```plaintext
    Name: HOST
    Secret: 个人服务器的IP地址
    ```
    ```plaintext
    Name: SSH_KEY
    Secret: 个人服务器的SSH私钥
    ```

### 3. 在个人服务器添加公钥
- 登录个人服务器，执行命令
    ```bash
    nano ~/.ssh/authorized_keys
    ```
- 粘贴公钥，按`Ctrl+O`保存，按`Ctrl+X`退出

### 4. 修改deploy脚本
- 打开博客项目下的`.github/workflows/pages-deploy.yml`文件
- 在末尾加入
    ```yaml
    # 同步到服务器
    sync-to-server:
        runs-on: ubuntu-latest
        needs: build
        steps:
        - uses: actions/checkout@v4

        - name: Download built site
            uses: actions/download-artifact@v4
            with:
            name: github-pages
            path: ./_site

        - name: Deploy to your server
            uses: burnett01/rsync-deployments@master
            with:
            switches: -avz --delete
            path: ./_site/
            remote_path: /home/ubuntu/blog # 个人服务器上的博客目录
            remote_host: ${{ secrets.HOST }}
            remote_user: ubuntu
            remote_key: ${{ secrets.SSH_KEY }}

        # 自动解压并清理压缩包
        - name: Extract artifact on server
            uses: appleboy/ssh-action@master
            with:
            host: ${{ secrets.HOST }}
            username: ubuntu
            key: ${{ secrets.SSH_KEY }}
            script: |
                cd /home/ubuntu/blog 
                if [ -f artifact.tar ]; then
                tar -xvf artifact.tar
                rm artifact.tar
                fi
            # cd /home/ubuntu/blog  # 个人服务器上的博客目录
    ```

### 5. 提交同步
- 提交修改到github仓库，等待github actions完成同步
